# SyndeoCare Design System

Welcome to the **SyndeoCare Design System** — the single source of truth for building consistent, accessible, and scalable healthcare interfaces.

## 🏗️ Architecture (DTCG Standard)

Our design tokens follow the **W3C Design Tokens Community Group** specification:

| Layer | Purpose | Example |
|-------|---------|---------|
| **Primitives** | Raw values | `color.green.500`, `spacing.4` |
| **Semantics** | Intent-based | `action.primary`, `feedback.error` |
| **Components** | Component-specific | `button.size.lg`, `card.padding` |

## 📚 Structure

- **Foundations** — Colors, Typography, Spacing, Shadows
- **Components** — Button, Card, Badge, Input, Form
- **Patterns** — Forms, Loading States, Navigation
- **Accessibility** — WCAG 2.2 compliance, ARIA patterns
